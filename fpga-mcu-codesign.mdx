An FPGA is brilliant at parallel, timing-critical work and clumsy at the flexible, sequential logic a microcontroller handles with ease. Most platforms teach FPGA in isolation, which is why few engineers can answer the question that matters on the job: where should this part of the system live? This lesson answers it and then connects the two worlds. #fpga #mcu #systemdesign

## Learning Objectives

By the end of this lesson, you will be able to:

1. **Choose** between an FPGA, a microcontroller, and a DSP for a given task.
2. **Explain** system-on-chip concepts and what a soft core is.
3. **Partition** a system across an FPGA and an MCU sensibly.
4. **Build** an SPI link between an FPGA accelerator and an STM32 or ESP32.

:::tip[Instrument it]
The interface between two chips is where integration bugs hide, so make the handshake visible. Put a logic analyser across the SPI bus and confirm the MCU command and the FPGA response match your protocol byte for byte. Expose a status register in the FPGA (busy, error, result-ready) that the MCU polls. That same register is a natural telemetry source: the [MCU can stream](/education/iot-systems/) the accelerator throughput or error count onward so the health of the combined system is [visible on a dashboard](https://siliconwit.io), not just on the bench.
:::

## Application Questions and Solutions

<InArticleAd />


### Question 1: Should this go on the FPGA or the MCU?

A design needs to timestamp incoming pulses to within 10 nanoseconds and also format the results into JSON for a web request. How would you partition it?

<details>
<summary>**Click to reveal the solution**</summary>

<Steps>

1. **Separate the two needs.** Nanosecond timestamping is hard real-time and parallel. JSON formatting and networking are flexible, sequential, library-heavy tasks. ✅

2. **Put timing on the FPGA.** Capture and timestamp the pulses in hardware, where the timing is deterministic and independent of any software load. ✅

3. **Put formatting on the MCU.** Hand the timestamps to the microcontroller over SPI, and let it build the JSON and handle the network, where a rich software stack already exists. ✅

</Steps>

</details>
