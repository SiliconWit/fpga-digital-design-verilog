You already use UARTs, timers, PWM channels, and CPU cores every time you program a microcontroller. This course teaches you to build them. You will describe digital hardware in Verilog, prove it works in simulation, run it on a real FPGA, and finish by pushing a design through the open-source ASIC flow to a manufacturable layout. #fpga #verilog #digitaldesign

## What You Will Learn

<CardGrid>
  <Card title="Describe hardware, not software" icon="open-book">
    Write Verilog that synthesises into real gates and flip-flops. Understand the difference between combinational and sequential logic, blocking and non-blocking assignments, and why hardware is parallel by nature.
  </Card>

  <Card title="Verify before you build" icon="open-book">
    Write self-checking testbenches and read waveforms, so a design is proven correct in simulation before it ever touches a board.
  </Card>

  <Card title="Run it on real silicon" icon="open-book">
    Take a Verilog design through synthesis, place and route, and timing on a low-cost FPGA using a fully open toolchain.
  </Card>

  <Card title="Build the blocks inside an MCU" icon="open-book">
    Create the peripherals you normally take for granted, a UART, a PWM generator, an SPI master, and even a small CPU, from scratch in hardware.
  </Card>
</CardGrid>

## Lessons

<CardGrid>
  <Card title="Lesson 1: Verilog Fundamentals" icon="star">
    [Start here](./verilog-fundamentals). Modules, ports, wires and registers, combinational versus sequential logic, and your first testbench.
    **Practical:** recreate logic gates and a 4-bit adder in Verilog and simulate them.
  </Card>

  <Card title="Lesson 2: Simulation and Testbenches" icon="document">
    [Open lesson](./simulation-and-testbenches). Why you simulate first, testbench structure, stimulus, self-checking assertions, and waveform analysis.
    **Practical:** write a self-checking testbench and inspect signals in GTKWave.
  </Card>

  <Card title="Lesson 3: State Machines in Verilog" icon="document">
    [Open lesson](./state-machines-verilog). Finite state machines, Moore versus Mealy, state encoding, and the clean two-block coding style.
    **Practical:** a traffic-light controller and a protocol-decoder FSM.
  </Card>

  <Card title="Lesson 4: First Design on a Real FPGA" icon="document">
    [Open lesson](./first-fpga-design-ice40). FPGA architecture (LUTs, flip-flops, block RAM), the open toolchain, pin constraints, and timing.
    **Practical:** blink and a debounced counter on an iCE40 board.
  </Card>

  <Card title="Lesson 5: Building MCU-Style Peripherals" icon="document">
    [Open lesson](./building-mcu-peripherals). PWM generators, UART transmit and receive, SPI, and shift registers in pure hardware.
    **Practical:** a UART transmitter and a PWM LED dimmer running on the FPGA.
  </Card>

  <Card title="Lesson 6: Memory, FIFOs, and Clock-Domain Crossing" icon="document">
    [Open lesson](./memory-fifos-clock-domain-crossing). Inferring block RAM, building FIFOs, metastability, synchronisers, and crossing clock domains safely.
    **Practical:** a dual-clock FIFO passing data between two clock domains.
  </Card>

  <Card title="Lesson 7: Building a Mini CPU" icon="document">
    [Open lesson](./building-a-mini-cpu). Datapath, ALU, register file, program counter, and instruction decode for a tiny instruction set.
    **Practical:** an 8-bit CPU that executes a small program from memory.
  </Card>

  <Card title="Lesson 8: FPGA plus MCU Co-Design" icon="document">
    [Open lesson](./fpga-mcu-codesign). When to choose FPGA versus MCU versus DSP, system-on-chip concepts, soft cores, and interfacing an FPGA to an MCU.
    **Practical:** an FPGA accelerator that talks to an STM32 or ESP32 over SPI.
  </Card>

  <Card title="Lesson 9: From FPGA to ASIC" icon="document">
    [Open lesson](./fpga-to-asic-sky130-openlane). The ASIC flow from RTL through synthesis, floorplan, place and route, to GDSII, using the Sky130 PDK and OpenLane.
    **Practical:** take an earlier Verilog block all the way to a manufacturable layout.
  </Card>
</CardGrid>

## Prerequisites

:::note[Before you start]
This course assumes you understand digital logic at the level of the [Digital Electronics and Logic](/education/digital-electronics/) course: number systems, logic gates, Boolean algebra, flip-flops, and counters. We do not re-teach those here. We start from how to describe that logic in Verilog and take it all the way to real hardware. Basic comfort with a command line and a text editor is enough on the software side.
:::
